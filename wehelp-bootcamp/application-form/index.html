想要加入的軟體公司，為什麼

在篩選目標公司時，我是往兩個方向去思考：

  1. 公司規模
  2. 產品走向

1. 公司規模

小型或新創公司通常員工較少，因此每個人需要負責的範圍較廣，需要學習更多的技能。同時，因為決策者較少，所以也會有更多參與決策的機會。

而中、大型公司有更多的員工，能夠學習的對象也更多。此外，中、大型公司在工作的分工通常較細，每個人負責的事情也相對單純，對於新人上手工作來說相對友善。

在面對第一份工作時，我會希望先踏穩腳步後穩定地成長與學習，所以中型公司對我可能是比較好的選擇。因為分工較細，工作上負責的事情相對單純，讓我比較能夠專注於自己負責的領域。

2. 產品走向

我也希望在一家有新產品、功能不斷創新改進的公司工作。這樣能夠不斷地學習和改進，也會有更多的機會成長。

關於這份申請網頁，開發時的技術心得

透過實際製作簡單的靜態網頁，可以初步理解 HTML 標籤的意義和 CSS 元素的應用，並實作一些簡單的 JavaScript 功能。

在這個過程中遇到的一些重點與應用，我把它列舉在下方。

HTML 架構：

練習設計網頁的基本架構，各種標籤的使用方法和代表意義，並在實際製作中運用。

例如：<header>、<nav>、<section>、<main>、<footer>等標籤。

CSS 樣式設計：

學習如何調整元素的屬性，以實現網頁外觀的設計和特效。

例如：元素的淡入淡出效果、偽元素的畫面裝飾等。

JavaScript 運用：

運用基本語法處理簡單的 DOM 操作，完成所需的功能和效果。

例如：自製按鈕來達成想要的滑動效果、隱藏與顯現等。

RWD 設計：

針對不同的介面大小，來設計適合的網頁外觀和使用體驗。

例如：在桌機上使用 hover 效果來設計按鈕，在手機上則使用 click 效果，並進行不同的網頁排版設計等。
